## Phase 1: HTML 骨格 + CSS 基盤（Tokens/Theme/Foundation/Layout/Utility）

### 確定した設計判断

- **reset.css の !important**: spec §4 に foundation.reset 例外の相互参照が追加済み（spec#56）。reset サブレイヤー内の !important は準拠対象外
- **reset.css vs base.css の境界**: kiso.css を参照元とし、日本語正規化系（text-spacing-trim, text-autospace, line-break 等）は reset に残す。`touch-action: manipulation` は `:root` 一括ではなくボタン等のインタラクティブ要素のみに適用（kiso.css と同じ判断）
- **tokens/ease.css 分離**: spec SHOULD のカテゴリ別ファイル分割に従い、structure.css から分離。Phase 2 で z が出たら tokens/z.css も追加
- **Perfect Fourth 比率**: コメントと実値が不一致だったため、不正確なコメントを削除。実値は要件定義テーブル準拠
- **ダークモード**: select 矢印・チェックマークの SVG data URI 内ではCSS 変数が使えないため、@media (prefers-color-scheme: dark) で別色を指定。技術的制約による正当な逸脱
- **CUSTOMIZE コメント**: form action、canonical URL、theme-color HEX、本文中のサービス名を網羅。画像には Retina 2x 注記付き
- **3 ファイル共通注記**: ヘッダー/フッターが 3 ファイルに重複しているため「変更時は全ファイルに反映」の NOTE コメントを配置
- **対象ブラウザ**: style.css 先頭に Chrome 123+, Safari 18+, Firefox 128+ と明記
- **@property フォールバック**: property.css に未対応ブラウザでの挙動注意コメントを配置
